2.第二版–实现tail -f

tail -f默认先读取最后10行数据,再从文件尾部读取实时数据.如果对于小文件,可以先读取所有文件内容,并输出最后10行:

但是,当文件很大的时候是不行的,特别是有些日志文件经常有几个G,全读出来内存就爆了,但是又没有其他接口可以指定指针跳到倒数10行,只能利用一些方法来模拟输出倒数10行:

  • 预估一行数据的字符长度,最好偏多,这里我按1024字符长度为一行来处理
  • 利用seek的方法,跳转到seek(-1024 * 10, 2)的字符
  • 如果跳转的字符长度大于原来文件长度,那就把所有文件内容打印出来
  • 利用换行符计算已取字符长度共有多少行,如果行数大于10,那只输出最后10行,如果只读了4行,则继续读6*1024,直到读满10行为止

此外,打印完后要处理游标,防止漏打或者重复打印数据.了解完后就可以开始重构read_last_line函数了.

3.第三版–优雅的读取输出日志文件

如果每秒读一次文件,实时性就太慢了,把间隔改小了,则处理器占用太多,如果能得知文件更新再进行,那性能就能得到保障了.庆幸的是,在Linux中inotify提供了这样的功能.
此外,日志文件有一个特点就是会进行logrotate,如果日志被logrotate了,那我们就需要重新打开文件,并进一步读取数据.这时候也可以利用到inotify,获取到文件重新打开的事件时,我们就重新打开文件,再读取.
